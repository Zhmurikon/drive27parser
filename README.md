# DRIVE27 автозапись

> Потому что вручную обновлять страницу расписания в 3 ночи — это не жизнь.

Монитор свободных слотов на вождение для автошколы на базе [app.dscontrol.ru](https://app.dscontrol.ru).  
Автоматически проверяет наличие свободных занятий и — если нашёл подходящий — **сам записывается**.

---

## Что это делает

- Авторизуется на сайте автошколы под вашим аккаунтом
- Каждые N минут проверяет расписание на ближайшие X дней
- Фильтрует слоты по дням недели и времени (настраивается)
- Фильтрует по конкретному инструктору (опционально)
- При нахождении подходящего слота — **автоматически записывается**
- Отправляет уведомление в Telegram через [telegram_application](https://github.com/Zhmurikon/telegram_application) (опционально)
- Соблюдает дневной лимит записей (чтобы не записаться на 5 занятий на один день)
- Умеет останавливаться после первой успешной записи — или продолжать мониторинг

## Установка и запуск

### 1. Клонируй репозиторий

```bash
git clone https://github.com/Zhmurikon/drive27parser
cd drive27parser
```

### 2. Создай `.env` файл

```bash
cp .env.example .env
```

Открой `.env` и заполни:

```env
# Логин и пароль от app.dscontrol.ru
DS_LOGIN=123456789
DS_PASSWORD=ВАШ_ПАРОЛЬ

# Leads API для уведомлений в Telegram (можно заменить на свой webhook)
LEADS_API_URL=http://localhost/api/v1/leads/
LEADS_API_KEY=apikey
LEADS_NAME=Монитор вождения

# Как часто проверять (секунды) и на сколько дней вперёд смотреть
CHECK_INTERVAL=120 # 2 минуты
LOOK_AHEAD_DAYS=14 2 недели

# ID инструкторов через запятую (пусто = любой)
# TEACHER_IDS=160823

# Записываться автоматически (true) или только присылать уведомление (false)
AUTO_BOOK=true

# Максимум записей в день
DAILY_BOOK_LIMIT=1

# Остановить мониторинг после первой успешной записи
STOP_AFTER_BOOK=false
```

### 3. Настрой расписание

В `.env` укажи, в какие дни и в какое время искать слоты:

```env
SLOT_MONDAY=No
SLOT_TUESDAY=17:45-22:00
SLOT_WEDNESDAY=No
SLOT_THURSDAY=17:45-22:00
SLOT_FRIDAY=No
SLOT_SATURDAY=08:00-17:00
SLOT_SUNDAY=No
```

`No` — день не подходит, пропускать.  
`17:45-22:00` — искать слоты, начинающиеся в этом диапазоне.

### 4. Запусти

```bash
docker-compose up -d --build
```

Логи:

```bash
docker-compose logs -f
```

## Переменные окружения

| Переменная | Описание | По умолчанию |
|---|---|---|
| `DS_LOGIN` | Логин на сайте автошколы | обязательно |
| `DS_PASSWORD` | Пароль | обязательно |
| `LEADS_API_URL` | URL для уведомлений | необязательно |
| `LEADS_API_KEY` | API ключ | необязательно |
| `CHECK_INTERVAL` | Интервал проверки, сек | `120` |
| `LOOK_AHEAD_DAYS` | Горизонт поиска, дней | `14` |
| `TEACHER_IDS` | ID инструкторов | пусто (любой) |
| `AUTO_BOOK` | Автозапись | `false` |
| `TIMESHIFT` | Сдвиг часового пояса в минутах | `-600` |
| `DAILY_BOOK_LIMIT` | Лимит записей в день | `1` |
| `WEEKLY_BOOK_LIMIT` | Лимит записей в неделю (0 = нет ограничений) | `0` |
| `STOP_AFTER_BOOK` | Стоп после записи | `false` |
| `SLOT_MONDAY` … `SLOT_SUNDAY` | Расписание поиска | `No` |

## Как это работает внутри

Сайт использует ASP.NET антифоргери-токены по схеме **двойного cookie**: один токен живёт в cookie, второй — парный — в скрытом `<input>` на HTML-странице. Сервер проверяет, что оба соответствуют друг другу.

Поэтому перед каждой записью монитор:
1. Авторизуется и получает session cookie
2. Загружает страницу расписания и вытаскивает парный CSRF-токен из HTML
3. Отправляет POST с session cookie + CSRF-токен в заголовке
